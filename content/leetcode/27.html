Question

Given an integer array {{% inline-code %}}nums{{% /inline-code %}} and an integer {{% inline-code %}}val{{% /inline-code %}}, remove all occurrences of {{% inline-code %}}val{{% /inline-code %}} in {{% inline-code %}}nums{{% /inline-code %}} in-place. The relative order of the elements may be changed.

Since it is impossible to change the length of the array in some languages, you must instead have the result be placed in the first part of the array {{% inline-code %}}nums{{% /inline-code %}}. Return {{% inline-code %}}k{{% /inline-code %}} after placing the final result in the first {{% inline-code %}}k{{% /inline-code %}} slots of {{% inline-code %}}nums{{% /inline-code %}}.

Do not allocate extra space for another array. You must do this by modifying the input array in-place with O(1) extra memory.

Usage

{{% code makefile %}}Input: nums = [3,2,2,3], val = 3 Output: 2, nums = [2,2,_,_] Input: nums = [0,1,2,2,3,0,4,2], val = 2 Output: 5, nums = [0,1,4,0,3,_,_,_]{{% /code %}}

Solution

{{% code java %}}public int removeElement(int[] nums, int val) { int i = 0; int n = nums.length; while (i < n) { if (nums[i] == val) { // replace current element with element from end of array nums[i] = nums[n - 1]; // reduce array size by one n--; } else { // we don't move on to the next element if we replace it i++; } } return n; }{{% /code %}}
