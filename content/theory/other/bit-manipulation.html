Two's Complement

Integers are usually represented using the two's complement representation. The leftmost bit is set to 1 to indicate the number is negative and 0 to indicate it is positive.

Calculating the two's complement negative notation of an integer you start with the absolute value of the number in binary and then invert the digits and add one. To convert back from two's complement notation, we once again invert and add 1 to obtain the absolute value of the number. For instance, for -30 we have:

{{% code text %}}00000000 00000000 00000000 00011110 11111111 11111111 11111111 11100001 // invert 11111111 11111111 11111111 11100010 // add 1 00000000 00000000 00000000 00011101 // invert 00000000 00000000 00000000 00111110 // add 1{{% /code %}}

An advantage of two's complement is that it unifies addition and subtraction. For instance, we have 69 - 12 = 69 + (-12) so:

{{% code text %}} 0000 0000 0000 0000 0000 0000 0100 0101 // 69 + 1111 1111 1111 1111 1111 1111 1111 0100 // -12 0000 0000 0000 0000 0000 0000 0011 1001 // 57{{% /code %}}
