--- title: "Binary Search" date: 2021-11-26 draft: false ---

Binary search finds the position of a target value in a sorted array. The algorithm begins by comparing the target value to the element in the middle of the array. If the target value matches the element, we are done. If the target value is less than the element, then search continues in the lower half of the array. Otherwise, the search continues in the upper half of the array.

Searching for Exact Match

We proceed using two pointers called lo and hi. Initially, lo is set to the first element of the array and hi is set to the last element. At each iteration, mid is calculated as ⌊(hi + lo) / 2⌋ and the element at that element is compared to the target. If a match occurs, we are done and mid can be returned. Otherwise, if the element is greater than the target we know to continue the search in the lower half as so set hi to mid - 1. Similarly, if the element is less than the target we set lo to mid + 1.

Searching for Closest Match

Suppose we have some condition to check and are after the closest matching element in the array. For instance, we might want to find the greatest element less than or equal to the target. Now, instead of iterating until the pointers cross, we stop only it no longer holds that lo is strictly less than hi. Also, one pointer, say lo, will be advanced to mid rather than to mid + 1. This way, our lo pointer always points to the closest match and so we return lo after finishing the iterations.

Complexity

The time complexity of binary search is \(O(1)\) in the best case and \(O(\log n)\) in the average and worst-case. The worst-case space complexity is \(O(1)\).
