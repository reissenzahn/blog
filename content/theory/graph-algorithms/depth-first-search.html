--- title: "Depth First Search" date: 2021-11-24 draft: false ---

Depth-first search is a graph traversal algorithm which explores a graph to find every vertex reachable from a starting vertex. The algorithm proceeds by repeatedly exploring an unvisited vertex adjacent to the current vertex. This continues until a dead-end is reached at which point the algorithm backtracks until it finds an unvisited vertex to explore. This process is repeated until the algorithm backtracks past the source vertex.

Recursive Approach

Depth-first search lends itself to an intuitive recursive implementation:

  1. Mark the current vertex as visited.
  2. Consider each of its adjacent vertices.
  3. For each adjacent vertex, if that vertex has not been visited then recurse with that vertex.

Above we start at A and explore B, C, D, G, L and P. We then backtrack to L and explore H. We backtrack to G and explore J, I, E and F. Again, we backtrack to I after which we explore M, N, K and O before backtracking back to A.

Iterative Approach

Depth-first search can be implemented iteratively using a stack. We start by marking the source vertex as visited and push it onto the stack. We then pop a vertex off the stack and consider each of its adjacent vertices. If a vertex adjacent to the popped vertex has not been visited then mark it as visited and push it onto the stack. We repeat this until the stack is empty.

Complexity

This has worst-case time complexity of O(n + m) where the graph has n vertices and m edges. The worst-case space complexity is O(n).
