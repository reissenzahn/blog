Height and Size

The height of a node is the number of nodes along the longest path from the node down to the farthest leaf node. The height of a tree is the height of its root. This can be determined recursively starting from the root and returning the maximum height of the two sub-trees + 1. The height of a null node is 0 though we instead return -1 in the base case as otherwise our result will be one greater than the height of the tree.

The size of a binary tree is simply the number of nodes in the tree. This can be calculated recursively starting at the root node. Each call returns (size of the left subtree) + (size of right subtree) + 1. The base case occurs when the current node becomes null at which point 0 is returned.

Comparing Binary Trees for Equality

To check if two binary trees are in fact the same tree we start at the root and recursively check if both the left child of the root equals the left child of the other root and likewise for the right children. This recursion will terminate when the nodes being compared are both null. An implementation will need to account for the case in which one child is defined and the corresponding child in the other tree is null in which case false should be returned.

Tree Traversals

  • Pre-order (VLR): visit the node then its left child and finally its right child (i.e. 4, 3, 8, 2, 5, 7, 6, 9, 1).
  • In-order (LVR): visit the left child then the node and finally its right child (i.e. 2, 8, 3, 5, 4, 7, 9, 6, 1).
  • Post-order (LRV): visit the left child then the right child and finally the node (i.e. 2, 8, 5, 3, 9, 1, 6, 7, 4).
