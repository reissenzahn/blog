Introduction

A hash table consists of an array of buckets that store key-value pairs. When inserting a key-value pair, a hash function is used to compute an index into the array of buckets and the value is stored in the corresponding bucket. Similarly, to lookup the value associated with a certain key, the hash function is used to compute the index of the appropriate bucket from which the corresponding value can be obtained.

Handling Collisions

A hash collision occurs when a hash function generates the same index for more than one key. In such cases, a collision resolution strategy must be used.

Common probe sequences include:

  • Linear probing: interval between probes is fixed (usually 1).
  • Quadratic probing: interval between probes is increased by adding the successive values of an arbitrary quadratic polynomial to the original hash index.
  • Double hashing: interval between probes is computed by a second hash function.
