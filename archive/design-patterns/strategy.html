Introduction

The Strategy Pattern is a behavioral design pattern that allows you to extract a family of algorithms into separate classes called strategies and to make their objects interchangeable. A context class can then delegate algorithm execution to a certain strategy object that is passed in by the client. This way the context becomes independent of concrete strategies, so you can add new algorithms or modify existing ones without changing the code of the context or other strategies.

Structure

  • The Context class maintains a reference to one of the concrete strategies and interacts with it via the strategy interface.
  • The Strategy interface declares the interface used to execute a strategy.
  • Concrete implementations of Strategy implement different members of a family of algorithms.
  • The context calls the execution method on the strategy object provided by the client each time it needs to run the algorithm.

Example

interface PaymentStrategy {

  void charge(double amount);
}

class CashStrategy implements PaymentStrategy {

  @Override
  public void charge(double amount) {
    System.out.println("Counting out cash...");
  }
}

class CardStrategy implements PaymentStrategy {

  @Override
  public void charge(double amount) {
    System.out.println("Swiping card...");
  }
}

class Store {

  private PaymentStrategy paymentStrategy;

  public Store() {
  }

  public void buy(double price) {
    this.paymentStrategy.charge(price);
  }

  public void setPaymentStrategy(PaymentStrategy paymentStrategy) {
    this.paymentStrategy = paymentStrategy;
  }
}

public class Example {

  public static void main(String[] args) {
    Store store = new Store();

    store.setPaymentStrategy(new CashStrategy());

    store.buy(5.0);
  }
}
