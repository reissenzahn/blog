Introduction

The Observer Pattern is a behavioral design pattern that lets you define a subscription mechanism to notify multiple observer objects about events that happen to a given subject object. This pattern allows you to create a one-to-many dependency between object without making the objects tightly coupled.

Structure

  • The Subject allows observers to subscribe to receive events or to unsubscribe to stop receiving events.
  • When an event occurs, the subject calls the update() method in the Observer interface on each observer object.
  • A ConcreteSubscriber performs some behavior in response to notifications received from the subject.
